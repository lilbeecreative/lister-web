Dev Workflow

How to make and deploy a change

1
Open Git Bash in VS Code
In VS Code, click the terminal dropdown → select Git Bash. Then switch to dev and get latest changes: git checkout dev && git pull origin dev
2
Edit your files
Make changes in VS Code. The main files are main.py for backend and anything in the templates/ folder for the UI.
3
Save and push to dev
Run this one line in Git Bash — replace the message with what you changed:
git add . && git commit -m "your message here" && git push origin dev
4
Test it
Wait ~2 minutes for Railway to deploy, then check the dev site to make sure everything works correctly.
5
Push to production
Only when confirmed working on dev. Run this one line to go live:
git checkout main && git merge dev && git push origin main && git checkout dev

Supabase
Database + photo storage. Tables: listings, listing_groups, group_photos, auction_research_sessions. Storage bucket: part-photos.
supabase.com

GitHub
Two repos: lister-web (main app) and mantle-scanner (background scanner worker). Always push to dev branch.

Google AI Studio

aistudio.google.com
GitHub Repos
📁
lister-web
Main app. FastAPI backend in main.py. HTML templates in /templates/. Static files in /static/. This handles the dashboard, auction scanner, CSV export, and all API endpoints.
github.com/lilbeecreative/lister-web
⚙️
mantle-scanner
Background worker in scanner_service.py. Polls Supabase for pending uploads, runs Gemini Vision to identify parts, creates listings with titles and pricing.
github.com/lilbeecreative/mantle-scanner
